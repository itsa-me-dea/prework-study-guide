HTML

Let's build a house! üõ†

  • The <html> represents the root (top-level element) of an HTML document, so it is also referred to as the root element. All other elements must be descendants of this element.
  • The <head> element contains information about the webpage.
    • The <meta> element represents metadata that cannot be represented by other HTML meta-related elements, like <base>, <link>, <script>, <style> or <title>.
    • The <link> element specifies relationships between the current document and an external resource. This element is most commonly used to link to stylesheets.
    • The <title> element defines the document's title that is shown in a browser's title bar or a page's tab.
  • The <body> element represents the visible content shown to the user.
    • The <header> element represents introductory content, typically a group of introductory or navigational aids. It may contain some heading elements but also a logo, a search form, an author name, and other elements.
    • The <main> element represents the dominant content of the of a document. The main content area consists of content that is directly related to or expands upon the central topic of a document, or the central functionality of an application.
      • The <section> element represents a generic standalone section of a document, which doesn't have a more specific semantic element to represent it.
      • The <h1> to <h6> elements represent six levels of section headings. <h1> is the highest section level and <h6> is the lowest.
      • The <img> element embeds an image into the document.
        • class can be used to define an alternate class for that particular element.
        • src specifies the path or URL to the image.
        • alt attribute specifies an alternate text for an image, if the image cannot be displayed.
      • The <ul> element represents an unordered list of items, typically rendered as a bulleted list.
      • The <ol> element represents an ordered list of items ‚Äî typically rendered as a numbered list.
      • The <li> element is used to represent an item in a list.
      • The <p> element represents a paragraph. Paragraphs are usually represented in visual media as blocks of text separated from adjacent blocks by blank lines and/or first-line indentation.
      • The <a> element (or anchor element), with its href attribute, creates a hyperlink to web pages, files, email addresses, locations in the same page, or anything else a URL can address.
        • href includes the URL that the hyperlink points to.
    • The <code> element displays its contents styled in a fashion intended to indicate that the text is a short fragment of computer code.
  • <!-- *insert comment* --> can be be used to add in code comments.
  • Check out this site for more info on HTML.

CSS

Let's paint the house!üè°

  • margin indicates how much space we want around the outside of an element (i.e. outside of the border).
  • padding indicates how much space we want around the content inside an element (i.e. inside of the border).

NOTE: margin-* and padding-* have additions that allow for more tuned applications (e.g. -top, -left, -bottom, -right, -block).

  • :not() can be used to apply a style to all elements except for what's in the parenthesis.
  • width defines the width of the element. At 100% it spans the whole webpage.
  • height defines the height of the element. Using auto allows the height to automatically scale according to the width and the amount of HTML content.
  • background sets all background style properties at once, such as color, image, origin and size, or repeat method.
    • background-color sets the background color of an element.
    • background-image sets one or more background images on an element.
    • background-position sets the initial position for each background image.
    • background-repeat sets how background images are repeated. A background image can be repeated along the horizontal and vertical axes, or not repeated at all.
    • background-size sets the size of the element's background image.
  • color sets the foreground color value of an element's text and text decorations.
  • font sets all the different properties of an element's font. This can be modified to address particular traits:
    • font-familyspecifies a prioritized list of one or more font family names and/or generic family names for the selected element. By providing a list of fonts, this provides a backup font in the case that any preceding fonts are not available through the user's browser.
    • font-size sets the size of the font.
    • font-style sets whether a font should be styled with a normal, italic, or oblique face.
    • font-weight sets the weight (or boldness) of the font.
  • text-align sets the horizontal alignment of the inline-level content inside a block element or table-cell box.
  • text-shadowadds shadows to text. It accepts a comma-separated list of shadows to be applied to the text and any of its decorations. Each shadow is described by some combination of X and Y offsets from the element, blur radius, and color.
  • animation applies an animation between styles, see more here. This is how it was applied on this site.
  • display sets whether an element is treated as a block or inline element and the layout used for its children, such as flow layout, grid or flex
  • object-fit sets how the content of a replaced element, such as an <img> or <video>, should be resized to fit its container.
  • object-position specifies the alignment of the selected replaced element's contents within the element's box.
  • overflow sets the desired behavior for an element's overflow ‚Äî i.e. when an element's content is too big to fit in its block formatting context ‚Äî in both directions.
  • bordersets an element's border. It sets the values of border-width, border-style, and border-color.
    • border-radius rounds the corners of an element's outer border edge. You can set a single radius to make circular corners, or two radii to make elliptical corners.
  • Check out this site for more info on CSS.

Let's see how to get started on our local repository: üíª

  1. Navigate to your folder with:

  2. Check what branch you are currently on by running git status
  3. If you are not on the main branch, run git checkout main to switch to the main branch
  4. Pull the remote main branch into the local main branch by running git pull origin main. Now your local and remote main branches are in sync!
  5. Create a new feature branch for the JavaScript file by entering the following into the command line:
    6. git checkout -b feature/add-changes
  6. Navigate to your desired folder in the project repo by using the cd command in the command line.
  7. You can create a new file inside your folder bu using the touch command in the command line. For example, use touch myfile.html (Note: instead of .html, you can also use .css, .js, .md, etc.)
  8. You can open VSCode with code . or open the specific file with start myfile.html (for Windows) or open myfile.html (for Mac).

Once we've made our edits, how do we merge our local into our remote repository? üê±‚Äçüíª

  1. In the terminal, navigate to the prework-study-guide directory. Use the following command to stage your changes:
    2. git add -A
  2. Next, commit your changes with a message that describes your changes:
    3. git commit -m ‚Äú<your message here>‚Äù
  3. Once the changes are added and committed locally, make sure the local branch is up to date before pushing up the changes. Pull in the base branch main to your feature branch feature/add-changes by running the following Git command:
    4. git pull origin main
  4. To complete the Git flow, use git push to push up your changes to the remote repository by running the following Git command:
    5. git push origin feature/add-changes

Now that the changes have been committed to the remote repository, we can merge the branch in GitHub. üë©‚Äçüíª

  1. Navigate to the prework-study-guide repository in GitHub. Change the branch to the feature/add-changes branch and refresh the page to confirm that the changes we pushed up are now in the feature branch in GitHub.
  2. Navigate to the "Pull Requests" tab and open a new pull request. Make sure that you select main as the base branch and feature/add-changes as the comparison branch. Click "Create pull request" to open a PR.
  3. Once on the pull request form, give your PR a title and a short description of the changes. Click the "Merge pull request" button to merge your work into the main branch.
  4. If this branch applied to an issue, we can go ahead and close the associated GitHub issue. Navigate to the "Issues" tab and select the related issue. Go ahead and close that issue.
